# 掼蛋游戏模块化构建计划

本文档用于指导在线好友房掼蛋项目的模块拆分和开发顺序。核心原则是：规则内核先行，Unity 表现层和 Photon 联机层后接入；规则代码保持纯 C#，方便自动化测试和断线重连。

## 1. 模块总览

建议第一版拆成 10 个模块：

1. Cards 牌基础模块
2. Rules 规则模块
3. GameState 对局状态模块
4. TurnFlow 回合流程模块
5. PlayerSeat 玩家座位模块
6. Bot 托管机器人模块
7. Serialization 状态序列化模块
8. Networking 联机模块
9. UI 表现模块
10. GameController 协调模块

## 2. Cards 牌基础模块

职责：

- 定义牌、花色、点数、大小王。
- 构建 2/3/4 副牌。
- 洗牌和发牌。
- 保证 4/6/8 人模式每人 27 张。

当前状态：

- 已实现 `Card`、`Suit`、`CardRank`、`JokerKind`。
- 已实现 `DeckBuilder` 和 `DealResult`。
- 已有基础测试覆盖。

## 3. Rules 规则模块

职责：

- 模式配置：4/6/8 人、牌副数、队伍。
- 当前级牌和红桃逢人配。
- 牌型识别。
- 出牌合法性判断。
- 压牌比较。
- 炸弹、同花顺、全王炸比较。
- 升级结算。
- 后续扩展进贡还贡。

短期重点：

- `HandPattern`：识别单张、对子、三张、三带二、顺子、三连对、钢板、普通炸弹、同花顺、全王炸。
- `PatternComparer`：比较同类牌型，处理炸弹压普通牌型，处理同花顺和全王炸等级。

## 4. GameState 对局状态模块

职责：

- 保存一局牌当前状态。
- 保存玩家手牌、当前轮最大牌、出牌历史、出完名次。
- 保存当前阶段：等待、发牌、进贡、还贡、出牌、结算。
- 保存当前等级、当前行动玩家、随机种子。

要求：

- 必须可序列化。
- 不直接依赖 Unity 组件。
- 后续作为 Photon 房间属性和断线重连的基础。

## 5. TurnFlow 回合流程模块

职责：

- 决定谁先手。
- 推进当前行动玩家。
- 处理出牌、过牌、一轮结束。
- 跳过已经出完牌的玩家。
- 判断一局是否结束。

边界：

- 不负责识别牌型。
- 不负责网络同步。
- 只调用 Rules 判断出牌是否合法。

## 6. PlayerSeat 玩家座位模块

职责：

- 管理座位、队伍、玩家身份。
- 支持 4/6/8 人交错组队。
- 记录玩家在线、准备、托管、断线状态。
- 记录房主和 Master Client 信息。

## 7. Bot 托管机器人模块

职责：

- 玩家断线或超时后自动操作。
- 第一版使用简单策略保证牌局能继续。

第一版策略：

- 能压则出最小可压牌。
- 不能压则过。
- 先手优先出最小单张、对子或合法牌型。
- 进贡交最大牌。
- 还贡交最小可还牌。

## 8. Serialization 状态序列化模块

职责：

- 将 GameState 转为 JSON。
- 从 JSON 恢复 GameState。
- 保存房间快照。
- 为断线重连和 Master Client 迁移做准备。

注意：

- 普通玩家的完整手牌不应明文公开给所有玩家。
- 第一版可以先在本地模拟中完整序列化，联机时再区分公开状态和私有手牌。

## 9. Networking 联机模块

职责：

- 登录 UserId 和昵称。
- 创建房间、加入房间。
- 准备、取消准备、开始游戏。
- 发送出牌、过牌、托管事件。
- 同步房间公开状态。
- 私发玩家手牌。
- 支持断线重连。

原则：

- Networking 只传事件和状态。
- 不在 Networking 内部重写规则。
- Master Client 调用 Core 规则模块执行权威计算。

## 10. UI 表现模块

职责：

- 登录界面。
- 大厅和好友房界面。
- 桌面、座位、手牌、出牌区。
- 出牌、过牌、托管、准备按钮。
- 倒计时和结算界面。

原则：

- UI 只展示状态和收集输入。
- UI 不直接决定规则结果。

## 11. GameController 协调模块

职责：

- 串联 UI、Rules、GameState、TurnFlow、Networking。
- 处理玩家输入命令。
- 调用规则验证。
- 更新状态并通知 UI 刷新。
- 联机模式下触发网络事件。

## 13. 推荐开发顺序

1. 完成 Cards 基础。
2. 完成 Rules 中的牌型识别和出牌比较。
3. 完成 GameState。
4. 完成 TurnFlow。
5. 做本地完整一局模拟。
6. 接入 Bot 托管。
7. 做状态序列化。
8. 做 Unity 桌面 UI 原型。
9. 接入 Photon 好友房。
10. 完善断线重连、托管和结算体验。
